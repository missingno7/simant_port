# boundary_heads.txt — HAND-CURATED env-wait loop heads (NE_SEG:HEX_OFFSET
# per line).  scripts/irgen.py reads this file AND the mechanically derived
# simant/facts/boundary_heads_derived.txt (scripts/waitscan.py — the
# enumeration of every pure wait loop; freshness suite-held via --check);
# each head is tagged BoundaryEffect in the IR, the emitter generates a
# host-side observer event + RESUME entries from it, and resume_calls goes
# pipeline-wide (the UNWIND RE-ENTRY RULE).
#
# A head must be a SEQUENTIAL/CALL instruction of the wait loop (never a
# jcc/jmp), with its successor inside the same function (the park resumes
# there).
#
# Interactive semantics (win16.interactive.InteractiveDriver.arm_boundary_parks):
# each pass through a head parks — unwinds the lifted Python chain, lets the
# CPU worker (or win16.callback.call_far's chunk loop, for waits reached
# inside a WndProc/TimerProc) advance the wall clock, and resumes INSIDE the
# lifted body at the RESUME entry, zero interpreted instructions.  Demo/
# headless runs leave cpu.boundary_hook None: the emitted observer is inert
# and the instruction-count timeline is unchanged.
#
# THIS file holds only loops waitscan classifies `mixed` (they also do light
# per-iteration work) that are PROMOTED BY EVIDENCE — a live/headless crash
# proving the loop is a wait, plus an oracle check that the lifted body is
# faithful (owner triage directive: lifter bug goes upstream, never into a
# fact).  Pure wait loops live in the derived file; do not duplicate them
# here.
#
# Oracle evidence held by all heads (hand + derived): the whole-demo
# api-aligned differential is byte-identical at all 39 checkpoints + final
# state on the observer-emitting graph (cold_nohooks exercises splash +
# intro + dialogs + pacing under the demo clock), and _WaitHundredths
# additionally by a direct per-call A/B (identical AX/DX/SP/flags/
# instr=50,011 over a 50-tick wait, headless clock).
